HotSpot JVM 主要参数解释

参数使用方式

  • -XX: +\
  • -XX: -\
  • -XX: \

查看默认参数

内存管理参数

参数 默认值 说明
DisableExplicitGC 关闭 忽略来自 System.gc() 触发的 GC
ExplicitGCInvokesConcurrent 关闭 使用 CMS 收集器收集 System.gc()
UseSerialGC Client模式默认开启，其它关闭 在Client模式下默认值，打开后使用 Serial+Serial Old 收集器组合
UseParNewGC 关闭 打开后，使用 ParNew+Serial Old 收集器组合
UseConcMarkSweepGC 关闭 打开后，使用 ParNew+CMS+Serial Old 收集器组合，如果 CMS 出现 Concurrent Mode Failure，Serial Old 作为后备收集器
UseParellelGC Server模式默认开启，其它模式关闭 在Server模式下默认值，打开后使用 Parallel Scavenge + Serial Old 收集器组合
ParallelGCThreads CPU核数 并行处理的线程数

CMS 收集器参数

参数 默认值 说明
UseConcMarkSweepGC 关闭 激活 CMS 收集器，默认Java8之前是并行收集器
UseParNewGC 关闭 年轻代使用多线程并发进行gc。当使用UseConcMarkSweepGC后，UseParNewGC会自动开启
CMSConcurrentMTEnabled 开启 并发的 CMS 阶段以多线程执行
ConcGCThreads 0 定义并发 CMS 过程运行时的线程数
CMSInitiatingOccupancyFraction
UseCMSInitiatingOccupancyOnly
CMSClassUnloadingEnabled 开启 CMS 收集器的增量模式
ExplicitGCInvokesConcurrent System.gc() 改为执行 CMS GC，而不是 Full GC
ExplicitGCInvokesConcurrentAndUnloadsClasses 当有系统 GC 调用时，永久代也被包括进 CMS 垃圾回收范围

多线程相关参数

参数 默认值 说明
UseSpinning Java6默认开启，Java5关闭 开启自旋锁避免线程频繁挂起和唤醒
PreBlockSpin 10 自旋锁的自旋次数
UseThreadPriorities 开启 使用本地线程优先级
UseBiasedLocking 开启 是否使用偏向锁
UseFastAccessorMethods 开启 频繁反射执行某个方法，生成字节码加快反射执行

性能参数

参数 默认值 说明
AggressiveOpts Java6默认开启，Java5默认关闭 使用激进的优化特性，具备正面和负面影响，需根据具体应用特点分析
UseLargePages 开启 使用大内存分页，需要 OS 支持
LargePageSizeInBytes 4MB 使用指定大小的内存分页，需要 OS 支持
StringCache 开启 是否使用字符串缓存

调试参数

参数 默认值 说明
HeapDumpOnOutOfMemoryError 关闭 发生 OOM 时是否生成堆转储快照
PrintGC 简单日志
PrintGCDetails 详细日志
PrintGCTimeStamps 时间戳，自 JVM 启动至今的时间戳
PrintGCDateStamps 绝对日期时间戳
loggc 日志输出到指定文件
